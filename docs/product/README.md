# 羽毛球活动预约系统 - 产品文档

## 1. 产品概述

### 1.1 产品背景

Sea_Feather 羽毛球俱乐部每周组织羽毛球活动，活动形式包括日常锻炼和比赛。为提升活动管理效率，减少人工统计工作，开发本预约系统。

### 1.2 产品目标

1. **活动发布自动化**: 管理员通过后台发布每周活动信息
2. **报名流程线上化**: 用户通过微信 H5 页面完成报名
3. **候补管理智能化**: 满员后自动进入候补，取消后自动递补
4. **积分体系化**: 活动支付转化为积分，用于商品兑换

---

## 2. 用户角色

| 角色 | 权限 |
|------|------|
| 普通用户 | 报名活动、查看已报名、查看积分、兑换商品 |
| 管理员 | 发布活动、管理成员、处理候补、管理商品 |

---

## 3. 业务流程

### 3.1 活动发布流程

```
管理员创建活动 → 设置时间/场地/价格 → 发布活动 → 用户可见
```

### 3.2 报名流程

```
用户查看活动 → 选择报名类型 → 确认报名 → 扣除积分 → 报名成功
                                    ↓
                              候补队列（满员时）
```

### 3.3 取消与候补流程

```
用户取消报名 → 释放名额 → 候补队列第一人自动递补 → 通知候补用户
```

### 3.4 积分流程

```
参加活动 → 支付积分（收入） → 积分到账 → 兑换商品 → 扣除积分（支出）
```

---

## 4. 功能详情

### 4.1 活动管理

#### 活动类型
- **比赛**: 正式比赛，有竞技性质
- **日常活动**: 自由练习，以锻炼为主

#### 场地说明
- 俱乐部共有 5 片场地：1号、2号、3号、5号、6号
- 发布活动时需选择占用的场地
- 用户可通过可视化图形查看场地占用情况

#### 价格设置
| 类型 | 说明 |
|------|------|
| 仅活动 | 仅参加羽毛球活动 |
| 活动+晚宴 | 活动结束后聚餐 |
| 仅晚宴 | 不打球，只参加晚宴 |

### 4.2 报名规则

1. 报名时需使用微信名称
2. 报名成功后扣除相应积分
3. 报名人数达到上限后自动进入候补
4. 取消报名后积分退还，进入候补的用户自动补位

### 4.3 候补机制

1. 候补人数上限由管理员设置
2. 候补按照时间顺序排列
3. 有取消时，候补第一人自动转为正式报名
4. 候补用户可随时取消候补

### 4.4 积分规则

1. 报名活动时支付多少积分，就获得多少积分
2. 积分可以累积，用于兑换商品
3. 取消报名时，积分全额退还
4. 积分不足无法报名

### 4.5 兑换商品

1. 管理员在后台添加商品（名称、图片、所需积分、库存）
2. 用户使用积分兑换商品
3. 兑换后扣除相应积分
4. 库存有限，先到先得

## 6. 异常情况处理

| 场景 | 处理方式 |
|------|----------|
| 积分不足 | 提示余额不足，无法报名 |
| 活动已满 | 自动进入候补队列 |
| 重复报名 | 提示已报名，不允许重复 |
| 管理员取消活动 | 通知所有报名用户，退还积分 |
| 活动已结束 | 关闭报名入口，显示已结束 |
